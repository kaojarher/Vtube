# VTuber 訪談系統 - 使用指南

## 系統概述

本系統是一個功能完整的 VTuber 訪談管理平台，整合了 Puter.js AI 對話生成與語音合成、Pollinations.ai 圖像生成等先進技術。用戶可以輕鬆創建、編輯和播放 VTuber 訪談內容。

## 核心功能

### 1. 角色管理

**創建角色**
- 點擊「新增角色」按鈕開啟角色編輯對話框
- 填入角色名稱、身份（主播或來賓）、描述
- 選擇角色的語音配置（語音引擎、語速、音高）
- 上傳或生成角色圖像

**編輯角色**
- 在左側邊欄的角色卡片上點擊「編輯」按鈕
- 修改角色的任何信息
- 點擊「保存」確認修改

**刪除角色**
- 在角色卡片上點擊「刪除」按鈕
- 確認刪除操作

**預設角色**
系統提供兩個預設角色：
- **小隻**（主播）：年輕活力，高音調，語速 1.2x
- **單單**（來賓）：沉著冷靜，低音調，語速 1.0x

### 2. 訪談創建與編輯

**創建新訪談**
- 點擊頂部的「新建訪談」按鈕
- 輸入訪談主題（例如：「燒仙草怎樣才比較好吃」）
- 選擇主播和來賓角色

**自動生成對話**
- 填入訪談主題
- 選擇主播和來賓
- 點擊「生成對話」按鈕
- 系統使用 Puter.js AI 自動生成 4-6 句對話

**手動編輯對話**
- 點擊對話框下的「編輯」按鈕
- 修改對話內容或更換角色
- 點擊「保存」確認修改

**添加對話**
- 點擊「新增對話」按鈕
- 系統會添加一個新的空白對話框
- 編輯並填入內容

**刪除對話**
- 點擊對話框下的「刪除」按鈕
- 確認刪除操作

### 3. 語音合成與播放

**合成單句語音**
- 在對話框下點擊「合成語音」按鈕
- 系統使用 Puter.js 的 txt2speech API 生成語音
- 語音參數根據角色配置自動應用

**合成全部語音**
- 點擊播放控制區的「合成語音」按鈕
- 系統依次為所有對話生成語音
- 每句語音生成後會自動保存

**播放單句語音**
- 在已合成語音的對話框下點擊「播放」按鈕
- 系統使用瀏覽器的 Web Audio API 播放語音

**播放全部訪談**
- 點擊「播放全部」按鈕
- 系統依次播放所有對話的語音
- 點擊「停止」可中斷播放

### 4. 圖像生成

**使用 Pollinations.ai 生成角色圖像**
- 選擇一個角色
- 點擊「生成角色圖像」按鈕
- 系統根據角色描述自動生成圖像
- 生成的圖像會自動保存到角色配置中

**上傳自訂圖像**
- 在「新增角色」對話框中填入圖像 URL
- 支援任何公開可訪問的圖像 URL

## 數據管理

### 匯出數據

**導出為 JSON 文件**
- 點擊頂部的「匯出」按鈕
- 系統會下載包含所有角色和訪談信息的 JSON 文件
- 文件名格式：`vtuber_interview_{時間戳}.json`

**JSON 文件結構**
```json
{
  "characters": [
    {
      "id": "char_host",
      "name": "小隻",
      "role": "host",
      "description": "年輕活力的主播",
      "imageUrl": "https://...",
      "voiceConfig": {
        "engine": "google",
        "voiceId": "en-US-Neural2-C",
        "language": "zh-TW",
        "speed": 1.2,
        "pitch": 1.5
      }
    }
  ],
  "interview": {
    "title": "訪談標題",
    "topic": "訪談主題",
    "dialogue": [
      {
        "characterId": "char_host",
        "characterName": "小隻",
        "text": "對話內容",
        "voiceUrl": null
      }
    ]
  },
  "exportDate": "2026-05-14T08:00:00Z"
}
```

### 匯入數據

**導入 JSON 文件**
- 點擊頂部的「匯入」按鈕
- 選擇之前導出的 JSON 文件
- 系統會自動加載文件中的所有角色和訪談信息

**本地存儲**
- 所有數據自動保存到瀏覽器的 LocalStorage
- 刷新頁面後數據不會丟失
- 清除瀏覽器數據會導致本地存儲的數據丟失

## 技術特性

### 集成的 API

**Puter.js AI 對話生成**
- 使用 GPT-4 模型自動生成自然流暢的對話
- 支援自訂提示詞以控制對話風格

**Puter.js 文字轉語音**
- 支援多種語音引擎（Google TTS 等）
- 可調整語速（0.5x - 2.0x）和音高（0.5 - 2.0）
- 支援多種語言和方言

**Pollinations.ai 圖像生成**
- 基於文字描述自動生成高質量圖像
- 無需 API 金鑰，直接調用
- 支援自訂圖像尺寸和風格

### 響應式設計

系統支援多種設備：

- **桌面版（≥1024px）**：三欄佈局，完整功能展示
- **平板版（768px - 1023px）**：兩欄佈局，可摺疊側邊欄
- **手機版（<768px）**：單欄佈局，垂直堆疊

### 性能優化

- 使用 Vanilla JavaScript，無框架依賴
- 本地 LocalStorage 存儲，快速訪問
- 異步 API 調用，不阻塞 UI
- 智能緩存機制

## 常見問題

### Q: 如何修改角色的語音？
A: 在角色卡片上點擊「編輯」，在「語音」下拉菜單中選擇新的語音選項，然後保存。系統提供四種預設語音選擇，分別對應不同的音調和性別。

### Q: 生成的對話不夠滿意怎麼辦？
A: 您可以手動編輯每一句對話。點擊對話框下的「編輯」按鈕，修改內容後保存即可。

### Q: 如何備份我的數據？
A: 點擊「匯出」按鈕下載 JSON 文件。建議定期備份重要的訪談數據。

### Q: 支援哪些語言？
A: 系統主要支援繁體中文（zh-TW）。語音合成可支援多種語言，具體取決於 Puter.js 的語音引擎支援。

### Q: 如何刪除所有數據？
A: 清除瀏覽器的 LocalStorage 數據即可。或者，您可以手動刪除每個角色和訪談。

### Q: 能否離線使用？
A: 不能。系統需要網絡連接以調用 Puter.js 和 Pollinations.ai 的 API。

## 隱私與安全

- 所有數據存儲在您的本地瀏覽器中
- 系統不會上傳個人信息到第三方服務器
- 使用 HTTPS 加密傳輸所有 API 請求
- 建議定期備份重要數據

## 技術支援

如遇到技術問題，請檢查：

1. 瀏覽器是否支援最新的 HTML5 和 JavaScript 特性
2. 網絡連接是否正常
3. Puter.js 和 Pollinations.ai 服務是否可用
4. 瀏覽器控制台是否有錯誤信息
